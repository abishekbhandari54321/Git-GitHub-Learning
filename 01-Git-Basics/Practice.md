# Practical Exercises

## Practical 1 - git init

### Command Used

```bash
git init
```

### Result

A hidden `.git` folder was created inside the project.

### Learning

`git init` converts a normal folder into a Git repository.

---

## Practical 2 - git status

### Command Used

```bash
git status
```

### Result

Git showed the current status of the repository and the untracked files.

### Learning

`git status` tells us what changes have been made in the repository.

---

## Practical 3 - git add

### Command Used

```bash
git add .
```

### Result

All files were added to the staging area.

### Learning

`git add .` prepares all changes for the next commit.

---

## Practical 4 - git commit

### Command Used

```bash
git commit -m "Initial commit: Added Git learning structure"
```

### Result

A new commit was created.

### Learning

A commit is a snapshot of the project at a particular point in time.

---

## Practical 5 - git remote add origin

### Command Used

```bash
git remote add origin https://github.com/abishekbhandari54321/Git-GitHub-Learning.git
```

### Result

The local repository was connected to GitHub.

### Learning

This command links the local repository with the remote repository.

---

## Practical 6 - git branch -M main

### Command Used

```bash
git branch -M main
```

### Result

The branch name was changed from `master` to `main`.

### Learning

This command renames the current branch.

---

## Practical 7 - git push

### Command Used

```bash
git push -u origin main
```

### Result

The local commits were uploaded to GitHub.

### Learning

`git push` sends local commits to the remote repository.
